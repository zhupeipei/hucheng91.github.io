接着上篇，经过技术组的讨论，目前采用前端页面全部用html，和java对应API打包在一个war里，前端开发人员只加载前端页面，调用服务器API，这样就避免了前端开发人员需要去理解jsp的，给Chrome设置可跨域模式，前端通过sessionStorage存储用户信息；配置全局的AJAX路径常量，配置全局页面调转路径；具体如下，Ajax路径和跳转路径

这2个路径用处非常大，在本地开发环境，前端只要将VIEWPATH配置成自己开发的服务器路径就好，等上传到服务器直接改成服务器路径，影响非常小，具体例子如下

就如同上面，只需要改变VIEWPATH ，前端开发人员就很容易根据自己本地环境自由配置，非常方便。登陆成功以后，就把这个用户信息存储到sessionStorage中，整个网站就可以共享这些信息。在用chrome启动时候配置可跨域模式即可，在其他页面调转也可以用seessionStorage判断，如果没有就不跳转。

但用seesionStorage会遇到一个问题，就是seesionStorage的生命周期并不是受后端控制，这就出了个麻烦事，比方重启服务器，后端的session没了，但seesionStora后端配置的session超时，前端并不能知道，这是个烦心事，最后，我通过在后端设置了一个监听器，监听session超时，然后在前端配置一个全局Ajax函数，具体如下

通过这个全局ajax函数，在每次请求ajax成功后拦截一下，看看sessionstatus的值，如果是”timeout”就清空sessionStorage，调转登陆界面。

这样就基本实现了，在开发的过程种有遇到了另外一个问题（这一路就各种踩坑啊），就是在跨域模式下，ajax请求并不能带上cookies，而后端是通过sessionID来判断seesion问题，后端存储的sessionId是传统的servlet，没有cookies，就不能识别，现在还没想好怎么解决，是通过类似微信的token或者将用户信息放到redis里，单独存储，缓存的键就用户的某个特有的信息比方数据库存储的，先想想这个问题在怎么解决。
