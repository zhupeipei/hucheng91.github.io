JavaScript 实现继承的几个关键字

  • 原型

  • 原型链

当eat方法被调用的时候，先在自己的方法列表中寻找，找不到，就去原型中，如果原型没有就去原型的原型，最后找到Object对象，Object都没有的话就是未定义了，这个找的过程其实就形成了一个原型链

以上这样，每次手动关联，是很烦的一件事，也不好理解，同时为了迎合当时如日中天的Java，对象是需要 new 出来的。

上面这个有个问题，每次 new 出来的对象都有一个sayHello函数，这样都得分点内存，要是很多对象的话，内存肯定吃不消，太浪费了！
然后 发明了一个原型对象（prototype）

这个所谓的构造函数Student其实是一个幌子，每次new Student的时候 ，确实会创建一个对象（blob，jack），并且把这个对象(blob或者jack)的(proto)指向了Student.prototype这个对象，这样就有了sayHello()方法；

原型（proto）和原型对象（.prototype）,就实现了JavaScript的面向对象，但是理解起来还是很麻烦的，不够优雅，加了下面的语法糖
